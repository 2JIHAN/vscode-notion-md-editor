# Notion Markdown Editor PRD

## 개요

Notion Markdown Editor는 Notion MCP, Notion API, 로컬 Markdown 동기화 워크플로우를 지원하는 VS Code 확장이다. 사용자는 Notion Enhanced Markdown에 가까운 문법으로 로컬 `.md` 파일을 편집하고, 변경된 파일만 Notion에 push 하여 MCP 사용량을 줄인다.

## 문제

현재 Notion MCP를 매번 사용해 페이지를 읽고 수정하면 토큰 사용량이 커진다. 또한 Notion callout 같은 Notion 전용 블록은 일반 Markdown editor에서 구조가 잘 보이지 않아 편집 경험이 떨어진다.

## 목표

- Notion 스타일 callout을 로컬 Markdown에서 쉽게 작성
- `.md` 파일을 Notion에 가까운 WYSIWYG 형태로 편집
- Notion sync용 `pull/`, `push/` 워크플로우와 호환
- 변경된 Markdown 파일만 Notion API 또는 MCP로 push 가능한 파일 구조 유지
- VS Code 기본 Markdown editor와 병행 사용 가능

## 비목표

- Notion 전체 기능을 그대로 복제하지 않음
- 완전한 Markdown round-trip 보장을 1차 목표로 삼지 않음
- Notion API sync 엔진을 이 확장 1차 버전에 포함하지 않음
- 데이터베이스 뷰, relation, rollup 편집 UI는 범위 밖

## 대상 사용자

- Notion 문서를 로컬 Markdown으로 편집한 뒤 다시 올리고 싶은 사용자
- Notion MCP 토큰 사용량을 줄이고 싶은 사용자
- Notion callout, heading, list, code 중심의 문서를 자주 작성하는 사용자
- VS Code 안에서 Notion 스타일 문서 초안을 작성하려는 사용자

## 핵심 시나리오

1. 사용자가 `notion-sync/pull/`에서 가져온 Markdown 파일을 연다.
2. `Notion Markdown: Open WYSIWYG Editor`를 실행한다.
3. WYSIWYG 화면에서 문단, 제목, callout을 수정한다.
4. 확장이 수정 내용을 원본 Markdown 파일에 저장한다.
5. 사용자가 변경된 파일만 Notion API 또는 MCP로 push 한다.

## 비기능 요구사항

- 별도 빌드 없이 Extension Development Host에서 실행 가능
- VS Code `TextDocument`를 문서 모델로 사용해 저장과 undo 흐름에 맞춤
- 웹뷰는 외부 CDN 없이 동작
- 사용자가 작성한 Markdown 파일을 가능한 한 사람이 읽기 쉬운 형태로 유지
- Notion sync 태그인 `<callout>` 문법을 보존

## Markdown 표준

Notion callout은 로컬 Markdown에서 아래 문법을 표준으로 사용한다.

```md
<callout icon="✅" color="green_bg">
**성공 기준**

업로드가 정상 완료된 상태
</callout>
```

## 성공 기준

- 사용자가 `.md` 파일을 WYSIWYG editor로 열 수 있음
- callout을 버튼으로 삽입할 수 있음
- WYSIWYG에서 입력한 내용이 `.md` 파일로 저장됨
- 저장된 callout Markdown이 Notion MCP Enhanced Markdown 또는 별도 Notion API 변환기에 재사용 가능

## 리스크

- `contenteditable` 기반 구현은 복잡한 Markdown 구조에서 손실 가능
- `document.execCommand`는 레거시 API이므로 장기적으로 ProseMirror, Milkdown, TipTap 같은 editor 엔진 검토 필요
- Markdown parser가 간단 구현이라 table, nested list, image, HTML block 처리 부족
- Notion API push 구현 시 Markdown과 Notion block 간 변환 규칙을 별도 모듈로 분리해야 함
