# Notion Markdown Editor Roadmap

## 방향

목표는 Notion sync용 Markdown을 VS Code에서 편하게 작성하고, 변경된 파일만 Notion에 올릴 수 있게 하는 것이다. 초기에는 WYSIWYG 편집 경험과 callout 안정성을 우선한다.

## Phase 0 완료

- VS Code 확장 기본 프로젝트 생성
- Markdown editor command 등록
- callout snippet 삽입
- callout preview webview
- WYSIWYG custom editor MVP
- 문서 작성

## Phase 1 편집 안정화

목표:

- Notion sync용 문서 작성에 필요한 기본 블록 안정화

작업:

- Markdown parser와 serializer 테스트 케이스 추가
- callout 내부 paragraph, list, code 처리 개선
- undo, redo 동작 확인
- 저장 타이밍과 dirty state 확인
- frontmatter 또는 metadata 보존
- table은 읽기 전용 표시라도 우선 지원

완료 기준:

- 기존 `notion-sync/pull/AI-workspace` 문서를 WYSIWYG로 열고 저장해도 핵심 구조가 유지됨
- callout round-trip 손실 없음

## Phase 2 Notion Sync 보조 기능

목표:

- 로컬 `pull/`, `push/` 워크플로우를 VS Code 명령으로 지원

작업:

- `Notion Markdown: Copy Pull File to Push` 명령
- `Notion Markdown: Show Changed Push Files` 명령
- `pull/`과 `push/` diff 표시
- 파일 상단 metadata에서 원본 Notion URL, page id 읽기
- 변경된 파일 목록을 quick pick으로 표시

완료 기준:

- 사용자가 변경 대상 파일을 VS Code 안에서 확인 가능
- push 대상 파일을 실수로 전체 선택하지 않도록 방지

## Phase 3 Notion API 연동

목표:

- MCP 사용량을 줄이기 위해 Notion API 기반 pull, push 지원

작업:

- `NOTION_TOKEN` 설정 방식 정의
- `AI_WORKSPACE_PAGE_ID` 설정 방식 정의
- page metadata fetch
- block children fetch
- Notion block to Markdown 변환
- Markdown to Notion block 변환
- 변경된 파일만 update
- 신규 파일 create

완료 기준:

- Notion API로 특정 page를 pull 가능
- 변경된 Markdown 파일만 Notion API로 push 가능
- Notion MCP 없이 기본 sync 가능

## Phase 4 WYSIWYG 엔진 교체 검토

목표:

- 직접 구현한 `contenteditable` 한계를 줄임

후보:

- ProseMirror
- TipTap
- Milkdown
- Lexical

검토 기준:

- Markdown round-trip 제어 가능성
- custom block 구현 난이도
- VS Code webview 번들 크기
- Notion callout 커스텀 노드 지원
- copy, paste 안정성

완료 기준:

- 현재 직접 구현보다 손실이 적은 editor stack 결정
- migration 비용과 이점 문서화

## Phase 5 패키징과 배포

목표:

- 로컬 개발용 확장을 설치 가능한 VSIX로 배포

작업:

- `@vscode/vsce` 설정
- extension icon 추가
- changelog 작성
- license 정리
- release build 스크립트 추가
- `.vsix` 패키징

완료 기준:

- `vsce package`로 VSIX 생성 가능
- Extension Development Host 없이 설치 가능

## 우선순위

1. callout round-trip 안정화
2. metadata 보존
3. diff UI
4. Notion API pull
5. Notion API push
6. WYSIWYG engine 교체
7. 패키징
