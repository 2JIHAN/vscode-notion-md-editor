이 문서는 개발 진행 이력, 의사결정, 현재 상태를 추적한다. 다음 작업자가 맥락을 다시 묻지 않고 이어서 개발할 수 있도록 유지한다.

## 2026-05-11

### 배경

Notion MCP로 매번 페이지를 읽고 수정하는 방식은 토큰 사용량이 크다. 사용자는 Notion 페이지를 로컬 Markdown으로 pull 한 뒤, Markdown을 수정하고, 변경된 파일만 push 하는 구조를 원했다.

### Notion sync 폴더 생성

`/Users/sonuvis-jihan/agent-workspace/notion-sync`를 생성했다.

초기 구조:

```text
notion-sync/
├── pull/
├── push/
├── archive/
└── README.md
```

운영 개념:

- `pull/` Notion에서 내려받은 원본 Markdown
- `push/` 수정해서 Notion에 올릴 Markdown
- `archive/` 이전 스냅샷 백업

### AI-workspace pull 미러 생성

Notion `AI-workspace` 페이지와 일부 하위 페이지를 Markdown으로 저장했다. 이후 사용자는 MCP보다 Notion API 기반 sync가 토큰 절감에 유리하다는 방향을 검토했다.

### Callout Markdown 표준 결정

Notion 스타일 callout은 순수 Markdown blockquote가 아니라 Notion Enhanced Markdown에 가까운 XML-like 태그를 사용하기로 했다.

표준:

```md
<callout icon="✅" color="green_bg">
**성공 기준**

업로드가 정상 완료된 상태
</callout>
```

### WYSIWYG 방향 전환

사용자가 WYSIWYG editor 형태를 원한다고 명확히 요청했다. 기존 preview 보조 방식에서 VS Code `customEditor` 기반 WYSIWYG MVP로 확장했다.

추가된 기능:

- `notionMd.wysiwygEditor` custom editor
- `Notion Markdown: Open WYSIWYG Editor` 명령
- `contenteditable` 기반 편집 화면
- toolbar 기반 H1, H2, Bold, Inline code, Callout 삽입
- DOM 편집 결과를 Markdown으로 직렬화해 원본 `.md`에 저장
