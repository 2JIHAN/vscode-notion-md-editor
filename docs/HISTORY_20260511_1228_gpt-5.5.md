BACKLOG 항목을 세 개 묶어 worktree 한 개씩 띄우고 subagent에 dispatch하는 흐름을 시험했다. `chore/lint-setup`, `feat/callout-backspace`, `feat/divider` 세 갈래로 진행 → code-reviewer 패스 → 지적 사항 반영 커밋 → rebase + ff merge. 머지 순서는 lint-setup을 먼저 깔아 두 feat 브랜치를 새 lint 룰 위에서 검증한 뒤 올렸다.

### ESLint, Prettier 설정 (BACKLOG F)

flat config `eslint.config.js`, `.prettierrc.json`, `.prettierignore` 추가. `eslint-config-prettier`로 충돌 룰 해제. `printWidth: 120`, `trailingComma: "none"`, `singleQuote: true` 등 기존 코드 스타일에 맞춰 튜닝해서 초기 도입 시 노이즈를 줄였다. `src/extension.js`, `test/markdown.test.js`는 일부 줄이 120 초과라 Prettier가 wrap. `src/markdown.js`의 미사용 인자 `openInfo`는 `_openInfo`로 리네임. `npm run lint`, `npm run format`, `npm run format:check` 스크립트 추가.

### 빈 callout Backspace 보호 (BACKLOG A)

WYSIWYG에서 빈 callout 본문에 커서를 둔 상태로 Backspace를 누르면 callout이 통째로 삭제되던 버그를 수정. `src/webview.js`에 `getCalloutContent`, `isCalloutBodyEmpty`, `isCursorAtStart`, `convertCalloutToParagraph` 보조 헬퍼를 추가했고, U+200B 정규식 `/​/g`로 zero-width space를 정상 strip한 뒤 빈 여부를 판단한다. 커서 위치 체크는 단순 `startOffset === 0`이 아니라 `range.startContainer`에서 callout content까지 거슬러 올라가며 매 단계가 "첫 의미 있는 자식"인지 확인하도록 보강했다. 빈 callout 확인 시 callout block 전체를 빈 `<p>`로 치환하고 caret을 그쪽으로 이동.

### 구분선 `---` 지원 (BACKLOG B P0)

parser, serializer, `renderHtml`, WYSIWYG 렌더와 autoformat을 한 번에 연결했다. `src/markdown.js`에서 `matchDivider(line)` 정규식 `^(-{3,}|\*{3,}|_{3,})\s*$`로 `---`, `***`, `___`를 모두 받아 `{ type: 'divider' }` AST 노드로 만든다. `collectParagraph`가 `matchDivider`에서 break하므로 `---`가 단락에 흡수되지 않는다. frontmatter는 `tryParseFrontmatter`가 `start === 0`에서만 분기하므로 문서 첫머리 `---` 충돌은 없다. 직렬화는 `'---'` 단일 형태로 정규화. WYSIWYG는 `runAutoformat`에서 `^(-{3,}|\*{3,}|_{3,})$` 패턴으로 paragraph를 `<hr>`로 교체하고 새 빈 paragraph로 caret 이동. CSS는 `var(--nme-border-auto)` 변수로 테마 대응. 회귀 테스트: paragraph + `---` (setext 미지원 lock-in), list + `---`, heading + `---`, `***`/`___` round-trip, frontmatter 가드.

### 노션 단축어 일괄 도입 (BACKLOG B P0, B P1, C)

저녁 세션. 노션 공식 도큐 + 트윗 + 외부 가이드 (Markdown Guide, Thomas Frank, Swimm) 를 리서치해 적용 가능한 단축어를 추렸다. 결정 사항: `> + space` 는 노션에선 토글이지만 우리는 토글 타입을 정식 도입하기로 했고, `" + space` 는 blockquote 로 매핑. `***` / `___` 는 자동변환에서 제거 (사용자 입력 의도 보존). `~~` 와 단일 `~` 둘 다 strikethrough 로 받지만 직렬화는 `~~` 로 통일.

**markdown.js 확장.** `todo` (체크박스) 와 `toggle` (`<details><summary>`) 두 신규 블록 타입을 AST 에 추가. `matchTodoItem` 정규식 `^[-*+]\s+\[([ xX])\]\s+(.*)$` 로 unordered list 보다 먼저 검사해서 `- [ ] foo` 가 일반 li 로 흡수되지 않게 했다. `collectToggle` 은 `<details>` 깊이 카운터로 중첩을 지원하고 첫 비공백 라인의 `<summary>...</summary>` 만 summary 로 인식, 나머지는 본문으로 흘려보낸다. 본문은 `parse()` 로 재귀 호출. 직렬화는 항상 `<details>\n<summary>...</summary>\n\n본문\n</details>` 형태. renderer 는 `<ul class="todo-list">` 와 `<details class="toggle">` 로 출력하며 `<input type="checkbox">` 와 `data-checked` 속성으로 상태 표현. inline 은 `renderInline` 에 `~~..~~` → `<del>`, `~..~` → `<del>` 패턴 추가. 회귀 테스트 11건 추가, 총 37건 통과.

**라인 첫머리 + space 자동변환.** `runAutoformat` 에 분기 묶음 추가. `# ~ ######` → `<h1>~<h6>`, `[-*+] ` → `<ul><li>`, `숫자. ` → `<ol><li>` (start 속성 유지), `[ x ]?` 박스 → `<ul class="todo-list">` + 체크박스 + 텍스트 span, `" ` → `<blockquote>`, `> ` → `<details>` + 선택 상태의 `<summary>`. ZWS 가 섞여 있을 수 있어 `replace(/​/g, '')` 후 정확 일치 비교. divider regex 는 `/^(-{3,}|\*{3,}|_{3,})$/` 에서 `/^-{3,}$/` 로 축소.

**인라인 감싸기 자동변환.** `tryInlineReplace` 헬퍼 도입. 기존 inline-code 처리와 같은 "prefix 보존 + 텍스트 노드 3분할" 방식으로 `**X**` → `<strong>`, `*X*`/`_X_` → `<em>`, `~~X~~`/`~X~` → `<del>`, `[X](url)` → `<a>` 를 처리. italic 패턴은 `(^|[^*])` 와 `(?!\*)$` 로 `**bold**` 와 충돌 회피. 링크는 `(^|[^!])` 로 `![](url)` 이미지 문법 배제.

**슬래시 메뉴 + 툴바 단순화.** 툴바를 `Saved` + `Light` 만 남기고 H1/H2/B/Code/Callout 드롭다운+버튼은 모두 제거. 빈 단락 안에서 `/` 입력 시 캐럿 위치에 검색 가능한 팝업 (13 항목: H1/H2/H3, 불릿/번호/체크/토글 리스트, 인용, 코드 블록, 구분선, 토글, 콜아웃 3종). 위/아래 화살표 탐색, Enter 선택, Esc/외부클릭/공백 시 닫힘. 선택 시 `/` + 필터 텍스트를 제거하고 `replaceOrInsert` 헬퍼가 빈 블록이면 교체, 내용 있으면 다음 줄에 삽입. `insertCallout` 의 `<select>` 의존을 제거하고 `insertCalloutKind(kindKey)` 로 분리.

**기타.** `Cmd/Ctrl+Shift+S` 를 strikethrough 토글로 바인딩 (Notion 호환). `nodeToMarkdown` 에 h4-h6 / `ul.todo-list` / `details` / `<del>` 직렬화 추가, `inlineMarkdown` 에 `<input type="checkbox">` 무시 처리 + `<del>/<s>` → `~~..~~` 변환 추가. CSS 는 `.todo-list` flexbox + checkbox 정렬, `.toggle` summary 들여쓰기, `del/s` 라인스루 스타일 추가. ghostdesk-vscode 컨테이너에서 Cmd+R 리로드 후 렌더링 시각 검증, 37/37 테스트 + lint 통과.
